# Contribution Guide

Thanks for wanting to contribute to Tubestack.

## How to contribute

1. Start by checking whether the idea, bug, or improvement already exists in the issues.
2. If it does not, open a new issue and describe the problem or feature clearly.
3. If you want to work on it, mention it in the issue first so it is not duplicated.
4. Fork the repo, make your changes, and keep the scope focused.
5. Open a pull request from your branch after the issue is discussed or approved.

## Contact

If you want to reach out directly, you can email me at [kabirkhanuja@gmail.com](mailto:kabirkhanuja@gmail.com).

## Before opening a PR

- Make sure the app still builds and runs.
- Keep the neo-brutalist style and existing UX direction intact unless the change is intentionally visual.
- Avoid unrelated refactors in the same PR.
- Add a short explanation of what changed and why.

## Good contribution ideas

- Bug fixes
- UI/UX improvements
- Accessibility improvements
- Performance tweaks
- Better empty states and onboarding
- Small quality-of-life features

## Things to avoid

- Large unplanned rewrites
- PRs without a clear issue or explanation
- Changes that break existing localStorage data without migration notes

## Notes

Tubestack is designed to stay lightweight, fast, and simple to use in the browser. Contributions that keep it clean and focused are the best fit.
